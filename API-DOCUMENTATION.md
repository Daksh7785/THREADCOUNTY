# ThreadCounty API Documentation

This document provides a comprehensive reference for the ThreadCounty backend API endpoints.

- **Base URL (Local)**: `http://localhost:5000/api`
- **Base URL (Production)**: `https://backend-sigma-liard-92.vercel.app/api`
- **Headers**: Authentication endpoints require `Authorization: Bearer <JWT_TOKEN>` where indicated.

---

## 1. Authentication Endpoints (`/auth`)

### Sign Up
Registers a new user account.
- **URL**: `/auth/signup`
- **Method**: `POST`
- **Body**:
  ```json
  {
    "email": "user@example.com",
    "password": "SecurePassword123",
    "name": "Jane Doe",
    "company": "Textile Labs Ltd"
  }
  ```
- **Response (201 Created)**:
  ```json
  {
    "message": "Registration successful. Please verify your email.",
    "profile": {
      "id": "u-user123",
      "email": "user@example.com",
      "name": "Jane Doe",
      "role": "user",
      "plan": "Free"
    }
  }
  ```

### Login
Authenticates a user and returns a JSON Web Token (JWT).
- **URL**: `/auth/login`
- **Method**: `POST`
- **Body**:
  ```json
  {
    "email": "user@example.com",
    "password": "SecurePassword123"
  }
  ```
- **Response (200 OK)**:
  ```json
  {
    "token": "eyJhbGciOi...",
    "user": {
      "id": "u-user123",
      "email": "user@example.com",
      "name": "Jane Doe",
      "role": "user",
      "plan": "Free"
    }
  }
  ```

## 2. User & Profile Endpoints (`/user`)
*All requests require a valid bearer token.*

### Get Profile
- **URL**: `/user/profile`
- **Method**: `GET`
- **Response (200 OK)**:
  ```json
  {
    "id": "u-user123",
    "email": "user@example.com",
    "name": "Jane Doe",
    "company": "Textile Labs Ltd",
    "role": "user",
    "plan": "Free",
    "storage_used": 154200,
    "created_at": "2026-06-27T01:23:45.000Z"
  }
  ```

### Update Profile
- **URL**: `/user/profile`
- **Method**: `PUT`
- **Body**:
  ```json
  {
    "name": "Jane Doe Updated",
    "company": "New Textile Mills"
  }
  ```
- **Response (200 OK)**: Updated user profile object.

## 3. Image Upload Endpoints (`/upload`)

### Upload Swatch Swatch
Uploads a fabric swatch image. Files are automatically compressed on-client before upload to ensure fast processing.
- **URL**: `/upload`
- **Method**: `POST`
- **Headers**: Requires JWT token and `Content-Type: multipart/form-data`.
- **Form Data**:
  - `file`: (File blob, allowed formats: JPG, JPEG, PNG, Max 5MB).
- **Response (201 Created)**:
  ```json
  {
    "message": "Image uploaded successfully.",
    "upload": {
      "id": "up-456",
      "user_id": "u-user123",
      "filename": "file-1782502122.png",
      "original_name": "cotton_macro.png",
      "file_size": 89400,
      "file_path": "api/upload/raw/up-456"
    }
  }
  ```

### Get Raw Swatch Image
Serves the raw binary image stream for rendering in image elements. No authentication required.
- **URL**: `/upload/raw/:id`
- **Method**: `GET`
- **Response (200 OK)**: Binary image stream (`image/png` or `image/jpeg`).

## 4. Analysis & Report Endpoints (`/report`)
*All requests require a valid bearer token.*

### Analyze Swatch
Triggers Gemini Vision AI (or heuristic fallback if API key is not set) to scan the upload and output structural details.
- **URL**: `/report/analyze`
- **Method**: `POST`
- **Body**:
  ```json
  {
    "uploadId": "up-456"
  }
  ```
- **Response (201 Created)**:
  ```json
  {
    "message": "Analysis completed successfully.",
    "report": {
      "id": "rep-789",
      "upload_id": "up-456",
      "user_id": "u-user123",
      "warp_count": 84,
      "weft_count": 81,
      "thread_density": 165,
      "fabric_type": "Cotton / Plain Weave",
      "confidence": 0.92,
      "suggestions": [
        "Warp and weft ratio close to 1:1, offering optimal tensile strength.",
        "Suitable for medium-weight shirts and home textiles."
      ],
      "created_at": "2026-06-27T01:25:00.000Z"
    },
    "aiPowered": true
  }
  ```

### List Reports
Returns all reports generated by the user.
- **URL**: `/report/list`
- **Method**: `GET`
- **Response (200 OK)**: Array of report objects with nested upload metadata.

### Delete Report
Deletes the report record and its associated upload files.
- **URL**: `/report/:id`
- **Method**: `DELETE`
- **Response (200 OK)**:
  ```json
  {
    "message": "Report and image file deleted successfully."
  }
  ```

## 6. Seeding Demo Data (`/demo`)
*Requires valid bearer token.*

### Seed Demo Swatches
Fills the account with 6 realistic fabric analysis samples.
- **URL**: `/demo/generate`
- **Method**: `POST`
- **Response (200 OK)**:
  ```json
  {
    "success": true,
    "message": "Fresh demo data successfully generated. Enjoy the test drive!"
  }
  ```

### Clear Demo Swatches
Removes all seeded demo objects from the account.
- **URL**: `/demo/clear`
- **Method**: `POST`
- **Response (200 OK)**:
  ```json
  {
    "success": true,
    "message": "All temporary demo data has been cleared."
  }
  ```
